i wrote the first version of the interpreter during the weekend. i hooked it on to the input element in the "play" page, so you can start playing with it. it doesn't interface with the game system yet, but that part should be relatively trivial.

the xor shell is a joy-like language. it's concatenative and functional. every function is unary, taking a stack as the only argument and returning a stack as the result.

there's pratically no syntax. the space between atoms denotes function composition. arguments always precede the function. you can build lists by putting words inside square brackets (`[` and `]`). lists and programs are the same thing. you can pass around programs just like you can pass around lists.
